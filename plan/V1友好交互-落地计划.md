# V1 友好交互与渲染升级 — 落地计划 v2

> 基于 DOM UI 迁移后的项目现状重写。
> 遵循现有 Simulation/Adapter 分离架构，增量改造，不破坏设计。

---

## 总体策略

分 **4 个阶段**，每阶段可独立验收。阶段间有轻微依赖（P2 依赖 P1 的分层，P3 依赖 P2 的进度数据），但每阶段内部的子任务粒度足够小，可逐步提交。

| 阶段 | 内容 | 核心改动区域 |
|------|------|-------------|
| P1 | 渲染分层 + 地表升级 + 树/物品视觉 | `render-sync.ts` |
| P2 | 平滑移动 + 朝向 + 任务进度条 + 邻格站位 | `movement.system.ts`, `render-sync.ts`, `main-scene.ts`, job 工厂 |
| P3 | DOM UI/HUD 改版 | `dom-ui-manager.ts`, `ui.css`, `index.html`, `presentation-state.ts` |
| P4 | 框选 + 区域 + 批量命令 | `input-handler.ts`, `world-preview.ts`, `presentation-state.ts` |

---

## 架构约束（贯穿所有阶段）

1. **Simulation 层不感知渲染**：`movement.system.ts`、`toil-executor.ts`、job 工厂等不引入任何 Phaser 或 DOM 依赖。朝向、插值等纯渲染概念不污染 `Pawn` 类型。
2. **Adapter 层单向读取 Simulation**：`render-sync.ts`、`dom-ui-manager.ts` 只读取 `World`/`GameMap`/`PresentationState`，不反向写入（除 `PresentationState` 这个桥梁）。
3. **PresentationState 是唯一桥梁**：所有"渲染层需要但不属于模拟"的瞬态数据（选中、悬停、工具模式、拖拽状态）都经由此接口。
4. **鸭子类型模式**：各系统通过窄接口读取 Pawn 数据（如 `MovablePawn`、`SelectablePawn`），不依赖完整 `Pawn` 类型。新增渲染数据时优先在 Adapter 层维护私有映射，而非扩展 Simulation 类型。
5. **DOM UI 不依赖 Phaser**：`dom-ui-manager.ts` 和 `ui.css` 不 import Phaser。UI 通过读取 `World`/`PresentationState` 获取数据。
6. **世界空间渲染留在 Phaser**：预览矩形、进度条、选中高亮等跟随摄像机缩放的视觉元素由 Phaser 渲染。

## P1：渲染分层 + 世界视觉升级

### 目标
- 地表不再是纯色方块，有颜色变化和边缘感
- 树底部对齐格子，幼苗到成树有大小变化
- 物品有独立的小图标渲染，可见堆叠数
- terrain 不再每帧重绘

### 1.1 RenderSync Container 分层

**当前问题**：所有 sprite 直接 `scene.add.*()` 加入场景根，仅靠 `setDepth()` 控制层序。没有逻辑分组，无法按层控制可见性，后续加遮罩/裁剪困难。

**改法**：在 `RenderSync` 构造时创建有序 Container 层：

```
depth 0  — terrainLayer      地形 (Graphics, dirty flag 控制重绘)
depth 1  — gridLayer         网格线 (Graphics, 只绘制一次)
depth 2  — designationLayer  指派标记
depth 3  — itemLayer         物品
depth 4  — plantLayer        植物/树
depth 5  — buildingLayer     建筑 + 蓝图 + 工地
depth 6  — pawnLayer         殖民者
depth 7  — worldUILayer      世界空间 UI (进度条、选中高亮、预览)
```

**具体改法**：

1. `RenderSync` 新增 `private layers: Map<string, Phaser.GameObjects.Container>` 和工厂方法 `private createLayers()`
2. `createSprite()` 根据 `obj.kind` 映射到对应 Container，用 `container.add(sprite)` 而非 `scene.add.*()` — sprite 仍通过 `scene.add` 创建（Phaser 要求），但立即 `container.add()` 接管归属
3. 移除 `createSprite` 和 `getDepth` 中的 `setDepth()` 调用——层序由 Container 的 depth 决定，Container 内部 sprite 无需独立 depth
4. `terrainGraphics` 和 `gridGraphics` 移入对应 Container
5. `worldUILayer` 暴露给 `WorldPreview` 和后续 P2 的进度条使用

**文件变更**：`src/adapter/render/render-sync.ts` — 重构

### 1.2 Terrain dirty flag

**当前**：`renderTerrain()` 每次 `fullSync()` 销毁重建 Graphics。`sync()` 不碰 terrain，但也没有地形变化时的增量更新机制。

**改法**：

1. 新增 `private terrainDirty = true` 字段
2. `fullSync()` 调用 `renderTerrain()` 后置 `terrainDirty = false`
3. 新增 `markTerrainDirty()` 公开方法
4. `sync()` 开头检查 `if (this.terrainDirty) { this.renderTerrain(); this.terrainDirty = false; }`
5. 在 `MainScene` 中监听 `designation_completed` 事件（mine 类型会改变地形），调用 `renderSync.markTerrainDirty()`

**文件变更**：
- `src/adapter/render/render-sync.ts` — 加 dirty flag
- `src/adapter/main-scene.ts` — 监听事件调 markTerrainDirty

### 1.3 地表视觉升级

**当前**：每个格子 `fillRect` 纯色（草地 `0x4a7c3f`，岩石 def.color）。

**改为**（纯 Graphics API，不用 shader/贴图）：

1. **基础色变体**：每格填充色在 def 基色上做 ±5% 随机偏移。用 `world.rng` 基于 `cellKey` 做确定性随机（同种子同地形同结果）
2. **边缘暗线**：每格底边 + 右边画 1px 暗线（基色 * 0.85），产生微凹凸感
3. **散落装饰**（仅草地）：约 12% 的格子画 1-2 个 2px 小圆点（深绿/棕色），模拟碎石草丛
4. **网格线**：`renderGrid()` 改为可开关的调试功能（受 `PresentationState` 的新字段 `showGrid` 控制），默认关闭。边缘暗线已替代网格的视觉作用

**新增到 PresentationState**：
```typescript
showGrid: boolean;  // 默认 false，F8 切换
```

**文件变更**：
- `src/adapter/render/render-sync.ts` — renderTerrain 重写
- `src/presentation/presentation-state.ts` — 加 showGrid
- `src/adapter/input/input-handler.ts` — 加 F8 快捷键

### 1.4 树木渲染修复

**当前问题**：三角形锚点在格子中心 `(cx, cy)`，视觉上树冠在格子正中，底部悬空。

**改法**：

1. y 锚点改为格子底部对齐：`cy = obj.cell.y * TILE_SIZE + TILE_SIZE`（底部边缘）
2. 三角形顶点重新计算：底边在格子底部，顶点向上延伸（树冠可超出格子上方）
3. 如果 plant 对象有 `growthProgress` 字段（0~1）：三角形整体按比例缩放，幼苗 50% → 成熟 100%。缩放通过调整三角形顶点坐标实现，不用 Phaser 的 setScale（避免模糊）
4. 颜色加确定性随机变体（基于 obj.id），避免所有树同色

**文件变更**：`src/adapter/render/render-sync.ts` — createSprite 中 Plant+tree 分支

### 1.5 物品渲染独立化

**当前**：物品和建筑等一样用 30x30 矩形渲染，视觉上无法区分。

**改为**：

1. 物品用 14x14 小矩形 + 1px 深色边框，居中在格子内
2. 堆叠数 >1 时：在 Container 内加一个 8px 白色文字标签显示数量，右下角偏移
3. 物品下方画一个 10x4 的深色椭圆（alpha 0.2）模拟投影

**实现**：物品的 `createSprite` 返回 Container（小矩形 + 数字标签 + 投影），类似 Pawn 的做法。`updateSprite` 更新数字标签。

**需要读取**：`(obj as any).stackCount` — 物品的堆叠数

**文件变更**：`src/adapter/render/render-sync.ts` — createSprite 中 Item 分支

- [ ] 地表有颜色变化，草地有散落装饰

## P2：平滑移动 + 朝向 + 任务进度条 + 邻格站位

### 目标
- Pawn 在格子间移动时平滑过渡，不再瞬移
- Pawn 执行任务时面朝目标方向
- 执行 Work/Wait/Interact 时头顶显示进度条
- 收割/建造像采矿一样站在邻格执行

### 2.1 平滑移动

需要三层协作：Simulation 保留信息 → MainScene 计算帧间 alpha → RenderSync 做插值。

#### 2.1a Simulation 层：保留 prevCell

**当前**：`movement.system.ts:101` 在移动时已计算 `prevCell` 但仅用于 `onObjectMoved` 后丢弃。

**改法**：在 `MovementState`（`movement.types.ts`）新增 `prevCell` 字段：

```typescript
interface MovementState {
  path: CellCoord[];
  pathIndex: number;
  moveProgress: number;
  speed: number;
  prevCell: CellCoord | null;  // 新增
}
```

在 `movement.system.ts` 移动到下一格时保存：
```typescript
// 现有代码已经有 const prevCell = { x: pawn.cell.x, y: pawn.cell.y };
// 只需加一行：
pawn.movement.prevCell = prevCell;
```

**注意**：`prevCell` 是 Simulation 数据（它记录"上一次实际移动前的位置"），不是渲染概念。这和朝向不同——prevCell 是客观事实。

相应修改鸭子类型 `MovablePawn`（`movement.system.ts` 内部接口）：
```typescript
interface MovablePawn {
  // ...现有字段...
  movement: {
    // ...现有字段...
    prevCell: { x: number; y: number } | null;  // 新增
  };
}
```

**文件变更**：
- `src/features/movement/movement.types.ts` — 加 prevCell
- `src/features/movement/movement.system.ts` — 保存 prevCell
- `src/features/pawn/pawn.types.ts` — movement 接口加 prevCell

#### 2.1b MainScene：计算 tickAlpha 传给 RenderSync

**当前**：`MainScene.update()` 调用 `this.renderSync.sync()` 无参数。

**改法**：

```typescript
// tick 循环结束后
const tickAlpha = this.accumulator / TICK_MS;  // 0~1，表示当前帧在两个 tick 之间的位置
this.renderSync.sync(tickAlpha);
```

暂停时不进入 tick 循环但仍调用 `sync`，此时 `accumulator` 为 0，`tickAlpha` 为 0，Pawn 停在格子中心——正确行为。

**文件变更**：`src/adapter/main-scene.ts` — sync 传参

#### 2.1c RenderSync：Pawn 位置插值

**改法**：`sync(tickAlpha: number = 0)` 签名变更。`updateSprite()` 增加 `tickAlpha` 参数。

对 Pawn 的位置计算：

```typescript
if (obj.kind === ObjectKind.Pawn) {
  const mv = (obj as any).movement;
  if (mv.path?.length > 0 && mv.pathIndex < mv.path.length && mv.moveProgress > 0) {
    // 正在两格之间移动：基于 moveProgress 插值
    const nextCell = mv.path[mv.pathIndex];
    const t = Math.min(mv.moveProgress, 1);
    const renderX = (obj.cell.x + (nextCell.x - obj.cell.x) * t) * TILE_SIZE + TILE_SIZE / 2;
    const renderY = (obj.cell.y + (nextCell.y - obj.cell.y) * t) * TILE_SIZE + TILE_SIZE / 2;
    sprite.setPosition(renderX, renderY);
    return;  // 跳过默认定位
  }
  // 静止或刚到达：定位到 cell 中心
}
```

**关键设计**：
- 插值基于 `moveProgress`（0~1），这是 Simulation 层每 tick 累加的值，天然表示"从当前格到下一格走了多远"
- 不使用 `tickAlpha` 做二级插值（过度平滑会导致位置预测错误，尤其在路径被阻断时）
- `moveProgress` 已被 `movement.system` 累加但尚未达到 1（未跳格），所以可直接作为插值参数
- 暂停时 `moveProgress` 不变，Pawn 停在当前插值位置——正确

**文件变更**：`src/adapter/render/render-sync.ts` — updateSprite 改造

### 2.2 朝向系统

**设计原则**：朝向是纯渲染概念，不属于 Simulation 层。不修改 `Pawn` 类型。

**实现方式**：在 `RenderSync` 内部维护一个 `facingMap: Map<ObjectId, Direction>`，在 `updateSprite` 中根据 Pawn 当前状态推导朝向：

```typescript
type Direction = 'up' | 'down' | 'left' | 'right';

private facingMap = new Map<ObjectId, Direction>();

// 在 updateSprite 中推导
private inferFacing(obj: MapObjectBase): Direction {
  const pawn = obj as any;
  const mv = pawn.movement;

  // 正在移动：根据当前格和下一格的方向差
  if (mv.path?.length > 0 && mv.pathIndex < mv.path.length) {
    const next = mv.path[mv.pathIndex];
    return this.directionFrom(obj.cell, next);
  }

  // 执行 Work/Interact：根据 toil.targetCell 和 pawn.cell 的方向差
  const job = pawn.ai?.currentJob;
  if (job) {
    const toil = job.toils[job.currentToilIndex];
    if (toil?.targetCell && (toil.type === 'work' || toil.type === 'interact')) {
      return this.directionFrom(obj.cell, toil.targetCell);
    }
  }

  // 保持上次朝向（或默认朝下）
  return this.facingMap.get(obj.id) ?? 'down';
}

private directionFrom(from: CellCoord, to: CellCoord): Direction {
  const dx = to.x - from.x;
  const dy = to.y - from.y;
  if (Math.abs(dx) > Math.abs(dy)) return dx > 0 ? 'right' : 'left';
  return dy > 0 ? 'down' : 'up';
}
```

**渲染表现**：在 Pawn Container 内的圆形边缘，用一个小三角形（5x5）指示朝向。三角形随朝向旋转。

**文件变更**：`src/adapter/render/render-sync.ts` — 新增朝向推导和渲染

### 2.3 任务进度条

**设计**：世界空间 UI，跟随 Pawn 移动，由 Phaser 渲染。

**实现**：

1. 在 `RenderSync` 中新增私有映射 `progressBars: Map<ObjectId, Phaser.GameObjects.Graphics>`
2. `createSprite` 创建 Pawn Container 时，同时创建一个隐藏的进度条 Graphics 加入 `worldUILayer`
3. `updateSprite` 中调用 `updateProgressBar(obj)`:

```typescript
private updateProgressBar(obj: MapObjectBase, sprite: Phaser.GameObjects.GameObject): void {
  const pawn = obj as any;
  const progress = this.getToilProgress(pawn);
  const bar = this.progressBars.get(obj.id);
  if (!bar) return;

  if (!progress) {
    bar.setVisible(false);
    return;
  }

  bar.setVisible(true);
  const spritePos = (sprite as any);  // Container 的 position
  bar.clear();
  const barW = 20, barH = 3;
  const x = spritePos.x - barW / 2;
  const y = spritePos.y - 20;  // 头顶上方

  // 背景
  bar.fillStyle(0x333333, 0.8);
  bar.fillRect(x, y, barW, barH);
  // 填充
  const ratio = progress.current / progress.total;
  bar.fillStyle(0x44cc44, 1);
  bar.fillRect(x, y, barW * ratio, barH);
}

private getToilProgress(pawn: any): { current: number; total: number } | null {
  const job = pawn.ai?.currentJob;
  if (!job) return null;
  const toil = job.toils?.[job.currentToilIndex];
  if (!toil || toil.state !== 'in_progress') return null;

  switch (toil.type) {
    case 'work':
      return { current: toil.localData.workDone ?? 0, total: toil.localData.totalWork ?? 100 };
    case 'wait':
      return { current: toil.localData.waited ?? 0, total: toil.localData.waitTicks ?? 60 };
    case 'interact':
      return { current: toil.localData.interacted ?? 0, total: toil.localData.interactTicks ?? 30 };
    default:
      return null;
  }
}
```

**文件变更**：`src/adapter/render/render-sync.ts` — 进度条渲染

### 2.4 邻格站位统一

**当前状态**：
- `mine-job.ts` ✅ 已有 `findAdjacentPassable()` → GoTo 邻格
- `harvest-job.ts` ❌ GoTo 直接到 `targetCell`（植物所在格）
- `construct-job.ts` ❌ GoTo 直接到 `siteCell`（工地所在格）

**改法**：

**Step 1**：提取 `findAdjacentPassable` 到公共模块：

新建 `src/features/ai/jobs/adjacent-util.ts`：
```typescript
import { CellCoord } from '../../../core/types';
import { GameMap } from '../../../world/game-map';

const ADJACENT_DIRS: CellCoord[] = [
  { x: 0, y: -1 }, { x: 0, y: 1 },
  { x: -1, y: 0 }, { x: 1, y: 0 },
];

export function findAdjacentPassable(target: CellCoord, map: GameMap): CellCoord | null {
  for (const dir of ADJACENT_DIRS) {
    const nx = target.x + dir.x;
    const ny = target.y + dir.y;
    if (map.pathGrid.isPassable(nx, ny) && map.spatial.isPassable({ x: nx, y: ny })) {
      return { x: nx, y: ny };
    }
  }
  return null;
}
```

**Step 2**：`mine-job.ts` 改为 import 公共版本，删除内部实现。

**Step 3**：`harvest-job.ts` 增加可选 `map` 参数：

```typescript
export function createHarvestJob(
  pawnId: ObjectId,
  targetId: ObjectId,
  targetCell: CellCoord,
  map?: GameMap,         // 新增
): Job {
  let gotoCell = targetCell;
  if (map) {
    const adj = findAdjacentPassable(targetCell, map);
    if (adj) gotoCell = adj;
  }
  // ... toils[0].targetCell = gotoCell（其余不变）
}
```

**Step 4**：`construct-job.ts` 同理增加可选 `map` 参数和邻格查找。

**Step 5**：`job-selector.ts` 调用处传入 `map`：
- L229: `createHarvestJob(pawn.id, desig.id, targetCell)` → `createHarvestJob(pawn.id, desig.id, targetCell, map)`
- L285: `createConstructJob(pawn.id, site.id, site.cell)` → `createConstructJob(pawn.id, site.id, site.cell, map)`

**不改的**：`PickUp`、`Drop`、`Deliver` 仍要求同格执行。`toil-executor.ts` 不改——Work toil 只检查 `workDone >= totalWork`，不检查 Pawn 是否在 targetCell 上。

### 4.2 InputHandler 拖拽状态机

**当前**：`input-handler.ts` 在 `pointerdown` 处理点击，无拖拽。

**改法**：新增拖拽检测，不改变现有单击逻辑的结构。

```typescript
// 新增私有状态
private dragState: {
  startScreenPos: { x: number; y: number };
  startCell: CellCoord;
  active: boolean;
} | null = null;

private static readonly DRAG_THRESHOLD = 6;  // 像素
```

**事件重构**：
- `pointerdown`：记录 `dragState = { startScreenPos, startCell, active: false }`
- 新增 `pointermove` 监听：如果 `dragState` 存在且距离超过阈值 → `active = true`，更新 `presentation.dragRect`
- `pointerup`（新增）：
  - 如果 `active` → 执行批量操作 → 清除 `dragRect`
  - 如果 `!active` → 执行单击（复用现有 `handleSelect/handleBuild/handleDesignate`）
  - `dragState = null`

**现有 `pointerdown` 中的单击逻辑移到 `pointerup` 的 `!active` 分支**——这是唯一的行为变化。

**文件变更**：`src/adapter/input/input-handler.ts`

### 4.3 框选行为矩阵

| activeTool | 单击（无拖拽） | 拖拽 |
|-----------|--------------|------|
| Select | 选中格子上的对象 | 框内所有对象加入 selectedObjectIds |
| Designate(mine) | `designate_mine` | 矩形内每个可采矿格子批量下发 |

### 4.4 拖拽预览渲染

**属于世界空间渲染**，在 `WorldPreview` 中新增。

拖拽过程中渲染：
1. **矩形边框**：白色虚线矩形覆盖拖拽范围
2. **格子高亮**：矩形内的每个合法目标格子用半透明色高亮

```typescript
// world-preview.ts 新增
private dragGraphics: Phaser.GameObjects.Graphics | null = null;

// 在 update() 中
if (presentation.dragRect) {
  if (!this.dragGraphics) {
    this.dragGraphics = this.scene.add.graphics().setDepth(50);
  }
  this.dragGraphics.clear();
  const { startCell, endCell } = presentation.dragRect;
  const minX = Math.min(startCell.x, endCell.x);
  const maxX = Math.max(startCell.x, endCell.x);
  const minY = Math.min(startCell.y, endCell.y);
  const maxY = Math.max(startCell.y, endCell.y);

  // 白色边框
  this.dragGraphics.lineStyle(2, 0xffffff, 0.8);
  this.dragGraphics.strokeRect(
    minX * TILE_SIZE, minY * TILE_SIZE,
    (maxX - minX + 1) * TILE_SIZE, (maxY - minY + 1) * TILE_SIZE
  );

  // 格子半透明填充
  this.dragGraphics.fillStyle(0xffffff, 0.1);
  this.dragGraphics.fillRect(
    minX * TILE_SIZE, minY * TILE_SIZE,
    (maxX - minX + 1) * TILE_SIZE, (maxY - minY + 1) * TILE_SIZE
  );
} else if (this.dragGraphics) {
  this.dragGraphics.clear();
}
```

## 风险控制

### R1：RenderSync 重构范围大
**策略**：P1 分两次提交：
1. 先只加 Container 分层 + dirty flag，不改 sprite 视觉，确认分层不影响现有渲染
2. 再改地表/树/物品的视觉表现

### R2：平滑移动与暂停/变速交互
**策略**：
- 暂停时 `moveProgress` 不变，Pawn 停在当前插值位置
- `Math.min(moveProgress, 1)` 钳制防止超出
- 变速只影响 tick 频率，不影响 `moveProgress` 的含义

### R3：DOM 按钮与 Phaser 键盘冲突
**策略**：
- 速度按钮 click handler 中调用 `(e.target as HTMLElement).blur()`
- 所有可交互 DOM 元素设置 `tabindex="-1"` 防止 Tab 聚焦
- `.tool-btn` 等用 `<span>` 而非 `<button>` 避免默认聚焦行为（需要时可升级为 button + blur）

### R4：框选性能
**策略**：
- 拖拽预览用单个 Graphics 批量绘制，不为每格创建 sprite
- 最大 50x50 截断
- `map.spatial.getAt()` 是 O(1) 查询，2500 次调用可接受
