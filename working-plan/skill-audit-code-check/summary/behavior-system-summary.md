# 行为系统 (Behavior System) 审计汇总报告

## 1. 现状总结 (Current State Summary)
行为系统在整体分层上基本遵循了 `oh-code-design` 的纯函数式、无状态突变（大部分）和无 UI 依赖的原则，但在模块职责边界、数据结构完整性以及具体决策维度的实现上存在较多缺口和偏差。

- **决策与评分 (Action Scorer & Context)**：当前的上下文汇总和行动评分过于简化。评分器缺失了设计文档中明确要求的“距离”、“当前携带状态”和“时间语境”等关键维度。上下文汇总器硬编码了小人的主状态为 `"idle"`，且未充分利用已有的时间事件和地图可达性信息。
- **状态机 (Behavior State Machine)**：状态机实现较为单薄，仅维护了平级的主状态（如 `eating`, `resting`, `working`），缺失了设计中要求的子状态（如伐木中、建造中）和目标对象引用。此外，状态转移的校验逻辑与文档声明的规则存在“双轨”漂移风险。
- **寻路与移动 (Pathfinding & Wander)**：A* 寻路和漫无目的散步的实现与地图子系统的“占用管理器联合查询”设计脱节。寻路算法未考虑动态占用和地形代价，且被过度耦合在行为规划模块中，加重了行为系统的职责。散步行为的实现（长路径随机）也与策划文档（相邻格随机）不符。
- **中断与调度 (Sim Loop & Goal Driven Planning)**：对工作被打断的逻辑处理不完整，仅实现了饥饿打断，遗漏了疲劳/休息的对称打断机制。同时，对“属于自己的床铺”等归属权逻辑缺乏校验。

## 2. 核心问题提炼 (Core Issues)

### 2.1 决策维度缺失与上下文不完整
- **行动评分器 (`action-scorer.ts`)** 未将小人与目标的距离、当前携带状态、昼夜时间语境纳入评分公式。
- **上下文汇总 (`behavior-context.ts`)** 未真实反映小人的当前主状态（硬编码为 `"idle"`），且未整合已认领的工单信息。
- **目标驱动规划 (`goal-driven-planning.ts`)** 在评估目标时未考虑携带物，且在寻找床铺时未校验“属于自己的床铺”这一归属约束。

### 2.2 状态机表达能力不足与规则漂移
- **状态机快照 (`behavior-state-machine.ts`)** 缺失子状态（如具体的工作阶段）和当前目标引用，无法完整表达设计文档中的“行为快照”。
- **状态转移规则** 存在代码运行时逻辑与 `DEFAULT_BEHAVIOR_TRANSITIONS` 文档化配置分离的问题，运行时并未真正校验配置表中的条件，易导致规则漂移。

### 2.3 寻路职责越权与占用逻辑脱节
- **寻路算法 (`a-star-pathfinding.ts`)** 放置在行为子系统中，违反了由地图系统提供“路径相关基础查询”的架构设计。
- **占用联合查询缺失**：寻路和散步规划（`wander-planning.ts`）均未与地图的“占用管理器”结合，忽略了他人的动态占用，且未处理起点不可走的情况。
- **散步逻辑不符**：散步规划采用了全局寻路选长路径的策略，与策划要求的“随机选择相邻地图格”相悖。

### 2.4 中断机制与配置泄漏
- **需求中断不完整 (`sim-loop.ts`)**：仅实现了饥饿紧迫对工作的打断，未实现疲劳/精力紧迫的打断逻辑。工单释放机制依赖外层调用方传入参数，存在静默失败风险。
- **配置泄漏 (`sim-config.ts`)**：模拟层配置中混入了 Phaser 主场景专用的常量（`MAIN_SCENE_NEED_GROWTH_SCALE`），破坏了行为系统与表现层解耦的原则。此外，核心模拟循环错误地依赖了 `headless` 目录下的类型。

## 3. 修改建议 (Refactor Suggestions)

### 3.1 完善决策与评分机制
- **丰富上下文输入**：修改 `aggregateBehaviorContext`，传入真实的 `BehaviorState`，并整合时间事件、完整的工作候选（含已认领）和地图可达性摘要。
- **升级评分公式**：在 `action-scorer.ts` 和 `goal-driven-planning.ts` 中引入距离衰减、携带物状态加权和昼夜时间语境加权，确保评分逻辑与设计文档完全对齐。
- **落实归属权校验**：在寻找休息目标时，接入实体/建筑子系统的归属数据，确保小人只前往“属于自己的床铺”。

### 3.2 升级状态机与对齐规则
- **扩展状态机快照**：在 `BehaviorFSM` 中增加 `subState`（子状态）和 `targetId`（目标引用），以完整支撑工作执行和需求满足的细粒度表达。
- **统一转移规则**：消除 `DEFAULT_BEHAVIOR_TRANSITIONS` 的双轨制，确保运行时的状态转移严格校验统一的规则表或谓词，并考虑提供不可变的快照转移 API 以支持重放。

### 3.3 重构寻路与地图交互
- **下沉寻路职责**：将 `a-star-pathfinding.ts` 迁移至 `src/game/map/` 目录，作为地图子系统的基础查询服务。解耦 `sim-loop.ts` 对 `headless` 目录的依赖，将相关 Trace 类型移至共享目录。
- **接入占用管理器**：在寻路和移动候选查询中，严格要求传入或联合 `OccupancyMap`，过滤被他人占用的格子，并增加对起点的合法性校验。
- **修正散步策略**：将 `wander-planning.ts` 和目标驱动中的散步逻辑修改为“相邻可行走/未占用格随机选择”，以贴合策划文档。

### 3.4 补全中断逻辑与清理配置
- **补全疲劳中断**：在 `sim-loop.ts` 中补充基于 `needs.rest`（疲劳/精力）的工作打断逻辑，并规范化中断原因的枚举。
- **解耦场景配置**：将 `MAIN_SCENE_NEED_GROWTH_SCALE` 等带有 Phaser 场景语义的配置移出 `sim-config.ts`，保持行为模拟参数的纯粹性。
