# 审计报告: src/scenes/renderers/tree-renderer.ts

## 1. 漏做需求 (Missing Requirements)

- [指控]: `oh-gen-doc/交互系统.yaml` 中 `交互反馈.视觉反馈.标记显示` 要求被标记的树木显示「对应的标记图标」。本文件在 `loggingMarked === true` 时仅调整树冠填充色、描边色与线宽（约 21–28 行），仍为矢量三角与矩形树干，**未**使用独立图标资源或与其他地图反馈项统一的「标记图标」呈现；若验收以「图标」字面为准，当前属于规格缩水或需策划确认「高亮树形」是否可等同图标。

- [依据]: `oh-code-design/UI系统.yaml` 中 `模块.地图叠加反馈层.职责` 明确包含「展示……标记图标……」，树木伐木标记的视觉语义应落在该层能力范围内或与之一致。

- [指控]: `oh-gen-doc/实体系统.yaml` 中树木 `伐木状态` 枚举含「正常 / 已标记伐木 / 伐木中」三态；`oh-code-design/实体系统.yaml` 中树木关键字段含「伐木标记」「是否被占用」。当前 `WorldEntitySnapshot` 对树仅暴露可选 `loggingMarked`（见 `entity-types.ts`），**无**与「伐木中」「是否被占用」一一对应的只读字段，故本渲染器无法区分「已排队标记」与「正在被小人执行」等状态；缺口主要在领域快照建模，但表现为地图层对策划三态反馈的缺失。

## 2. 无用兼容与 Mock (Useless Compatibility & Mocks)

- 未发现明显问题（无 mock、无 TODO、无临时兼容分支）。

## 3. 架构违规 (Architecture Violations)

- 未发现明显问题：函数仅接收 `Phaser.GameObjects.Graphics`、`WorldGridConfig` 与 `Iterable<WorldEntitySnapshot>`，通过 `g.clear()` 与绘制 API 输出像素，**不**回写世界或实体；与 `oh-code-design/实体系统.yaml` 中「读取投影层……为 UI、交互反馈、调试视图提供只读投影」及 `oh-code-design/UI系统.yaml` 中「以读模型驱动展示，避免 UI 直接承担领域规则」的方向一致，未见越权修改核心数据或绕过编排层。

## 4. 修复建议 (Refactor Suggestions)

- [行动点 #0346]: 若需严格对齐策划文案，在地图叠加反馈层为已标记树木增加与物资标记同体系的图标（Sprite/纹理）或复用现有任务标记绘制管线，与当前几何树形分工清晰（树形=资源体，图标=交互状态）。
- [行动点 #0347]: 若需表现「伐木中」或「被工作预占」，先在只读快照或叠加数据流中暴露与 `oh-gen-doc` 三态及 `oh-code-design`「是否被占用」对齐的字段，再在本渲染器或专用叠加层增加第三套样式，避免仅在呈现层猜测工作项状态。
- [行动点 #0348]: 将标记态与非标记态的颜色、线宽、透明度抽为常量或主题配置，便于与 `UI系统` 其他地图反馈样式统一验收（对比度、色盲友好等）。
