# 审计报告: src/data/grid-cell-info.ts

## 1. 漏做需求

- **[指控]**：格子悬停文案中的「地形」完全来自本地 `MOCK_BIOME_BY_KEY` / `MOCK_BIOME_ROTATE`，未从地图域的只读格属性或投影数据读取；与「地图格应承载基础空间属性、UI 以读模型展示」的目标不一致。
- **[依据]**：`oh-code-design/地图系统.yaml` 中「地图网格」职责为维护所有地图格的基础属性；「空间投影层」职责为向 UI 输出可用性等提示；核心数据「地图格」包含坐标、可通行状态、占用对象集合等字段。当前实现仅对可通行性调用了 `isWalkableCell`，未暴露或聚合真实地形/地块类型，也未展示占用信息。
- **[指控]**：不可通行时的文案写死为「障碍（mock 石块）」，未区分阻挡来源（配置中的阻挡格、建筑占用等），与「占用管理器记录阻挡」类需求未在悬停层面对齐。
- **[依据]**：同上，`oh-code-design/地图系统.yaml`「占用管理器」与核心数据「地图格」中的占用语义；当前文件只依赖 `WorldGridConfig` 的行走判定，未接入占用只读视图。

## 2. 无用兼容与 Mock

- **[指控]**：文件头与实现明确保留「mock 生物群系」及轮换字符串；第 35 行在不可通行分支中嵌入「mock 石块」文案，属于与设计目标并存的临时展示逻辑。
- **[影响]**：玩家看到的「地形」与地面上真实规则无关，容易在接入真实地图数据前形成错误产品预期；与 `src/data/ground-items.ts` 的 mock 散落物叠加后，整格信息多为演示数据而非单一真实来源。
- **[说明]**：注释称「未来接入真实地图数据时换掉实现」——在技能语义下仍归类为待拆除的 Mock/脚手架，而非已完成需求。

## 3. 架构违规

- **[指控]**：`src/data/grid-cell-info.ts` 从 `../game/map/world-grid` 引入类型与 `isWalkableCell`，形成 **data 目录对 game 模块的依赖**，与常见分层（游戏域可依赖通用数据/类型，而非反过来）相反，增加循环依赖与测试隔离风险。
- **[依据]**：`oh-code-design/UI系统.yaml` 强调「以读模型驱动展示」「界面状态层订阅领域系统只读数据并转成界面态」。将面向 HUD 的格式化逻辑放在 `src/data/`，同时反向依赖 `game/map`，边界上更接近「场景/UI 读模型」而非纯数据定义，与文档中的界面分层描述不够贴合。
- **[指控]**：悬停格式化与 `ground-items` 的 mock 数据源耦合，未通过地图或实体系统的统一只读接口聚合，与「来自地图系统与实体系统的可视对象状态」的输入边界（见 `oh-code-design/UI系统.yaml` 接口边界）仅为部分吻合。

## 4. 修复建议

- **[行动点 #0005]**：在地图或空间投影只读 API 中提供「格子的地形/标签 + 阻挡或占用简述」，令 `formatGridCellHoverText`（或其后继函数）仅消费该读模型，删除或缩小 `MOCK_BIOME_*` 与「mock 石块」类硬编码文案。
- **[行动点 #0006]**：将格悬停文案生成迁至 `src/scenes/`、`src/ui/` 或与「地图叠加反馈层」一致的模块，并把对坐标的类型与行走判定改为由上层注入接口或共享 `types`/「空间模型层」包，避免 `data` 依赖 `game`。
- **[行动点 #0007]**：地面掉落物与占用信息接入后，在悬停中统一从实体/地图只读查询拼接「掉落」行，替代对 `groundItemAt` 的静态 mock 依赖链（与 `ground-items.ts` 的演进一并规划）。
