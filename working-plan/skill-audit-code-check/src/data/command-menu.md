# 审计报告: src/data/command-menu.ts

## 1. 漏做需求

- **[指控]**：顶层菜单分组与策划文档不一致。`oh-gen-doc/UI系统.yaml` 与 `oh-gen-doc/交互系统.yaml` 规定主菜单层级为「区域 / 建造 / 家具」（区域下挂「存储区 → 新建」，建造下挂「墙 → 木墙」）。本文件将分类定义为 `orders`（展示为「指令」）、`structures`（「结构」）、`furniture`（「家具」），且「储存区」命令挂在 `structures` 下，而非独立的「区域」分支，未完整体现策划中的菜单树拆分。
- **[依据]**：`oh-gen-doc/UI系统.yaml` 中「菜单系统 → 主菜单结构」；`oh-gen-doc/交互系统.yaml` 中「菜单选择 → 菜单层级」。

- **[指控]**：建造路径的层级深度与文档不一致。策划为「建造 → 墙 → 木墙」三级；本文件以单条命令 `build-wall`（木墙）扁平表达，数据层未建模「墙」子菜单/子分组（若验收以文档结构为准，则属于结构简化或未落地的层级需求）。
- **[依据]**：同上，`oh-gen-doc/UI系统.yaml`「建造 → 墙 → 木墙」与 `oh-gen-doc/交互系统.yaml`「建造 → 墙 -> 木墙」。

- **[说明]**：`oh-gen-doc/UI系统.yaml` 将「物资拾取标记工具」与「伐木工具」描述为工具栏能力；本文件在「指令」类中提供 `lumber`、`haul` 等命令及热键映射。领域侧 `haul` 与拾取标记的对应关系需结合 `apply-domain-command` 等实现判断——本文件层面若仅对照文档条目名称，则「搬运」标签与「物资拾取标记」表述存在偏差风险，建议在验收时明确是否接受该命名映射。

## 2. 无用兼容与 Mock

- 未发现 `mock`、`temp`、`// TODO` 或明显仅为兼容旧系统而保留的死分支配置。
- `commandMenuDomainSemantics`（约第 240–249 行）在 `getCommandMenuCommand(commandId)` 失败时回退到 `defaultCommandMenuCommandId()`，属于防御性默认行为；在 `CommandMenuCommandId` 类型约束下正常调用路径不应触发，但若运行时有非预期 id 传入，可能掩盖错误。不宜归类为典型 Mock，但属于可讨论的「静默回退」兼容写法。

## 3. 架构违规

- **[指控]**：`oh-code-design/UI系统.yaml`「菜单模型」职责包括「管理区域、建造、家具等菜单层级」。本文件以 `CommandMenuCategoryId = "orders" | "structures" | "furniture"` 建模，与文档中的「区域 / 建造 / 家具」语义与边界不完全对齐（尤其「区域」与「建造」被合并进「指令 / 结构」），属于菜单模型与架构文档术语、分域不一致，易造成后续扩展（如单独增加「区域」下其它子功能）时继续偏离设计。
- **[依据]**：`oh-code-design/UI系统.yaml` 模块「菜单模型」职责与核心数据「菜单树」关键字段（菜单分组、子菜单、操作项、对应交互模式）。

- **[说明]**：`oh-code-design/UI系统.yaml` 扩展点写明「菜单树与工具栏可改为数据驱动配置」；本文件以数据形式集中定义命令、热键槽位与 `domainVerb`/`markerToolId` 映射，与「数据驱动」方向一致。未发现本文件直接修改运行时游戏状态或绕过交互系统边界的行为，不构成典型的分层越权实现。

## 4. 修复建议

- **[行动点 #0001]**：对照 `oh-gen-doc/UI系统.yaml` / `oh-gen-doc/交互系统.yaml`，将分类 id 与展示层级调整为「区域 / 建造 / 家具」（或等价映射并在 UI 层显式注释与文档的对应表），将 `storage-zone` 归入「区域」分支，将墙/蓝图类归入「建造」分支，减少与策划菜单树的心智偏差。
- **[行动点 #0002]**：若需严格三级菜单，在菜单数据或 `menu-model` 消费侧增加「墙」等中间分组，再挂接「木墙」命令，而不是仅扁平 `build-wall`。
- **[行动点 #0003]**：复核 `haul` 与策划「物资拾取标记」的语义与文案：若功能等价，在 `label`/`hint`（含 `villager-tools` 等与热键联动处）统一用语，避免玩家与验收文档理解不一致。
- **[行动点 #0004]**：评估将 `commandMenuDomainSemantics` 的回退改为开发期断言或显式错误上报，避免非法 id 静默落到默认命令。
