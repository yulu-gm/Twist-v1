# 审计报告: src/data/ground-items.ts

## 1. 漏做需求

- **[指控]**：`groundItemAt` 与 `MOCK_SCATTERED_GROUND_ITEMS` 用固定格坐标 + 任意 `displayName` 表示地面物资，未体现策划文档中的「物资实体」完整语义（类型枚举、可拾取状态、与存储区/携带关系等）。
- **[依据]**：`oh-gen-doc/地图系统.yaml` 中「地图初始化 → 初始内容 → 物资分布」要求初始物资为地图上散落状态、可被后续交互承接；`oh-gen-doc/实体系统.yaml` 中「物资」定义了**类型**（可选值含初始物资、木头、包装食品）、**位置**、**可拾取状态**等字段。本文件仅有 `displayName` 字符串与数量，且名称（如木柴、石块、浆果、绳结、铁矿、草药、兽皮）**未落在**上述物资类型枚举内，属于需求覆盖缺口或与文档未对齐的平行数据。
- **[指控]**：与 `oh-code-design/实体系统.yaml` 中「物资实体」关键字段（物资类型、所在容器类型/标识、可拾取标记等）及「实体查询投影 → 为地图显示提供稳定查询结果」相比，本模块未提供基于实体标识或统一类型系统的只读投影，而是第二套静态表。
- **[依据]**：`oh-code-design/实体系统.yaml` 核心数据「物资实体」「实体查询投影」；`oh-code-design/地图系统.yaml` 空间索引层职责「按坐标索引实体」——地面散落物作为地图上的实体/占用信息，应由该索引链路查询，而非独立 `Map` 硬编码。

## 2. 无用兼容与 Mock

- **[指控]**：文件头与导出名明确为 **mock**（`MOCK_SCATTERED_GROUND_ITEMS`），注释写明「未来接入真实库存系统时换掉」，属于技能所述「临时逻辑 / Mock 数据」长期驻留。
- **[影响]**：仓库内已有 `src/scenes/renderers/ground-items-renderer.ts` 从领域（WorldCore / `kind==="resource"` 等）同步地表物资展示，而 `grid-cell-info.ts` 仍通过 `groundItemAt` 读取本文件，易形成**两套地面叙事**（渲染与 hover 文案不一致），新系统未完全接管「格子上有什么」的展示与查询。
- **[说明]**：本文件本身不是「兼容分支」代码，但承担与真实实体快照并行的演示数据源，符合「无用兼容与 Mock」审计维度下的主要风险点。

## 3. 架构违规

- **[指控]**：在 `src/data/` 中维护与**领域地面物资**同构的权威查询入口（`GROUND_ITEMS_BY_KEY` + `groundItemAt`），绕开了 `oh-code-design/实体系统.yaml` 划分的**实体目录 / 实体查询投影**与 `oh-code-design/地图系统.yaml` 的**空间索引层**，使 UI 侧可通过静态表「假装」格上有物，而不必经过统一快照。
- **[依据]**：同上设计文档中分层与模块职责；另 `oh-code-design/实体系统.yaml` 风险项指出若实体位置与地图占用索引分开维护易出现不一致——本文件正是与领域实体并行的一套按格索引。
- **[次要]**：`src/data` 依赖 `../game/map/world-grid` 的 `coordKey` / `GridCoord`，将「策划/配置向」目录与具体地图网格实现耦合；若设计意图是 data 仅为无运行时依赖的静态配置，则边界偏模糊（需与项目约定对照，此处记为**边界异味**）。

## 4. 修复建议

- **[行动点 #0008]**：在 `grid-cell-info`（及任何仍调用 `groundItemAt` 的路径）改为只读消费与 `ground-items-renderer` **同源**的世界快照或「实体查询投影」结果（例如按格聚合 `containerKind==="ground"` 的物资实体），然后删除或缩小本文件中的静态散落表。
- **[行动点 #0009]**：若短期仍需演示数据，应将散落物生成迁入**初始化流程**（地图初始化 / 实体创建），使 `displayName` 或标签来自统一物资目录，并与 `oh-gen-doc/实体系统.yaml` 物资类型或可扩展表对齐，避免 hover 与模拟层各用一套名字。
- **[行动点 #0010]**：待无调用方后，移除 `MOCK_SCATTERED_GROUND_ITEMS` 与 `groundItemAt`，或保留为**测试/fixture 专用**文件并移出 `src/data` 生产路径，避免与领域查询并存。
