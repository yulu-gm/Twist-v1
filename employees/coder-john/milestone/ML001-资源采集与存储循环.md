# 里程碑 01：基础资源采集与存储循环

## 里程碑目标
实现游戏中最基础的资源获取与存储的闭环体验。玩家可以通过指令系统指挥小人获取环境中的自然资源，并将其统一收集到指定的存储区域中。

## 玩家体验场景
1. **划定存储区**：玩家可以通过左下角的指令菜单，选择“区域-存储区-新建”，然后在地图上框选出一片空地作为存储区。
2. **标记采集目标**：玩家可以通过指令菜单，使用“命令-伐木”、“命令-开采”或“命令-拾取”工具，在地图上框选树木、岩石或散落的物资。被选中的目标会在地图上显示出对应的标记状态。
3. **小人自动工作**：空闲的小人会自动发现这些被标记的工作目标，并前往执行：
   - 对于树木和岩石：小人走到目标旁边，头顶出现进度条进行采集（伐木/开采）。
   - 对于散落物资：小人走到物资所在格子，直接将其拾取并拿在手上。
4. **资源掉落与搬运**：
   - 伐木和开采进度完成后，树木和岩石会从地图上消失，并在原地掉落对应的资源（木头、石头）。
   - 小人会自动寻找地上散落的资源（包括刚掉落的和初始散落的），将其拾取。
   - 携带资源的小人会自动寻路前往玩家划定的“存储区”，并将资源放置在存储区的空闲格子上。
5. **资源汇总展示**：玩家可以直观地看到原本散落在地图各处的资源，被小人们一点点搬运并整齐地堆放在存储区内。

## 验收标准
- [ ] 玩家能够成功在地图上划定和取消存储区。
- [ ] 玩家能够成功框选树木、岩石、散落物资，并在地图上看到清晰的标记反馈。
- [ ] 小人能够正确寻路到被标记的目标处。
- [ ] 伐木和开采有明确的进度条反馈，完成后目标消失并正确掉落对应的资源实体。
- [ ] 小人能够将地上的资源拾取，并在UI或小人模型上表现出“携带资源”的状态。
- [ ] 小人能够将携带的资源准确放置到存储区内的空闲格子上，且同一格子只能放置同种资源（或遵循堆叠规则）。
- [ ] 当没有被标记的工作或存储区已满时，小人会恢复到漫无目的散步的待机状态。
