# 菜单与工具栏系统重构计划

## 1. 背景与目标
当前代码实现中，左下角的 `#command-menu` 包含了完整的一级、二级、三级菜单层级，而底部的 `#villager-tool-bar` 作为一个独立的组件，通过状态同步与左下角菜单保持一致。

根据最新的 `oh-gen-doc/UI系统.yaml` 和 `oh-code-design/UI系统.yaml` 设计，这是一种误解。正确的视觉与逻辑结构应该是：
- **左下角主菜单**：仅包含一级菜单（指令大类）和二级菜单（指令子类）。
- **底部工具栏**：作为三级菜单（具体工具）的容器。
- **交互联动**：玩家在左下角点击二级菜单项后，底部工具栏动态展开/显示对应的三级具体工具选项。

本计划旨在指导如何将现有代码重构为上述结构。

## 2. 数据模型重构 (`src/ui/command-menu.ts`)
当前 `DEFAULT_COMMAND_MENU` 包含了完整的树形结构。需要将其重构为两部分，或者修改现有的类型定义：
- **主菜单模型 (MainMenu)**：只包含 `Root` (一级) 和 `Second` (二级)。
- **工具栏模型 (Toolbar)**：`Second` 节点不再直接包含 `leaves` 或 `groups` 嵌套的 `leaves`，而是映射到一个“工具组 ID”。底部工具栏根据这个 ID 动态加载对应的工具列表。
- **状态管理**：`CommandMenuState` 拆分为 `MainMenuState` (记录展开的一级/二级) 和 `ToolbarState` (记录当前显示的工具组和选中的具体工具)。

## 3. HUD DOM 封装重构 (`src/ui/hud-manager.ts`)
- **左下角主菜单 (`#command-menu`)**：
  - 修改 `renderCommandMenu` 逻辑，使其只渲染到二级菜单。
  - 当二级菜单被点击时，不再在左下角展开三级选项，而是触发一个回调，通知系统“当前激活的工具分类已改变”。
- **底部工具栏 (`#villager-tool-bar`)**：
  - 废弃写死的 `VILLAGER_TOOLS` 静态渲染逻辑。
  - 提供一个新的方法 `renderToolbar(tools: ToolItem[])`，根据左下角选中的二级分类，动态清空并重新渲染底部的工具按钮。
  - 保留工具栏的快捷键绑定逻辑，但需要使其适应动态变化的工具列表。

## 4. 场景集成重构 (`src/scenes/GameScene.ts`)
- **状态桥接**：在 `GameScene` 中，监听主菜单的二级分类选择事件。当分类改变时，从数据源中获取该分类下的具体工具列表，并调用 `hud.renderToolbar()` 更新底部工具栏。
- **工具选择逻辑**：统一工具选择的入口。点击底部工具栏的具体工具时，执行原有的 `selectVillagerTool` 或 `selectCommandLeaf` 逻辑（进入对应的交互模式，如 `storage-zone-create`、`mine-mark` 等，并显示 `mode-hint`）。
- **默认状态**：游戏初始状态或点击“待机/取消”时，底部工具栏应恢复为默认的常用工具列表（或清空，取决于具体设计，建议保留基础的待机/取消按钮）。

## 5. 样式调整 (`index.html` & CSS)
- 移除左下角菜单中用于渲染三级选项（`.command-menu-children` 内的叶子节点）的特定样式，如果它们不再被使用。
- 确保底部工具栏 (`#villager-tool-bar`) 的样式能够适应动态数量的工具按钮，保持居中和横排展示。

## 6. 验证标准
- 运行 `npm run build` 无 TypeScript 和 Vite 构建错误。
- 浏览器内测试：点击左下角“指令” -> “区域”，左下角不再向下展开，而是底部工具栏变为“新建”等区域工具。
- 在底部工具栏点击“新建”，正确进入存储区创建模式，出现模式提示文本。
- 符合 `oh-acceptance/UI系统.yaml` 中的 `UI-001` 和 `UI-005` 用例。
