# PT010：伐木与开采工作的图形与 UI 表现（对齐 PT008）

## 需求依据（oh-gen-doc，仅列规格已写明的视觉项）

- `oh-gen-doc/行为系统.yaml` — **执行工作状态** 子状态：
  - **开采中**：行为为 **显示开采动画和进度条**
  - **伐木中**：行为为 **显示伐木动画和进度条**
- `oh-gen-doc/UI系统.yaml` — **状态反馈**：
  - **小人状态**：**小人动画表现（如工作、进食、睡觉）**；**状态图标（开采中、伐木中、搬运中、建造中等）** —— 本 PT **仅实现与 PT008 已落地的伐木/开采工作** 对应项：**工作向动画表现**、**开采中 / 伐木中** 状态图标。
  - **进度条**：**显示工作进行进度**；应用场景含 **开采读条进度、伐木读条进度**；**显示位置：目标实体上方**。

> 本计划 **不** 将建造读条、搬运图标、进食/睡眠动画等纳入范围（除非与伐木/开采读条可复用同一套进度条组件而不额外扩展话题）；**禁止** 为人工验收单独立项增加规格未提及的调试菜单、工单列表浮层、Registry 仅调试用覆盖层等。

## 现状与背景

- PT008 已在模拟层实现小人领取工单、走向目标、`perform-work` 与计时结算；**画面上** 仍难以区分「走向目标 / 读条工作中 / 空闲」等与规格一致的状态。
- 需在 **不改写需求** 的前提下，把上述 YAML 中的 **动画、进度条、状态图标** 接到现有 `PawnState` / 工作类型 / `actionTimer`（或等价进度来源）上，使表现与领域状态一致。

## 目标话题（单一）

为 **伐木、开采** 两类工作补齐需求文档要求的 **小人侧工作动画表现**、**开采中/伐木中状态图标**，以及在 **目标实体（树/岩石）上方** 的 **读条进度条**；并保证「无工作时」不误显工作进度条。

## 实现要点（建议方向，以代码现况为准）

1. **进度条（目标实体上方）**  
   - 当小人处于对应当前工单的读条执行阶段且类型为伐木/开采时，按 `UI系统` 在 **目标树/岩石格对应的实体或锚点上方** 绘制进度（与 `actionTimerSec`/总时长比例一致）。  
   - 工作完成、打断、离开时清除，避免残留在地图上。

2. **小人工作动画表现**  
   - 在现有小人渲染路径上区分「执行伐木 / 执行开采」可视状态（若当前为静态图块，可用规格允许的最低限度表现：如独立姿态帧、色偏、或占位循环，但须能明确读出「工作中」而非闲逛移动）。

3. **状态图标（开采中、伐木中）**  
   - 在小人附近或HUD约定位置显示与 **当前工作子类型** 一致的图标（具体美术可占位，但语义须与规格一致）。

4. **非范围（明确不做）**  
   - 建造读条、拾取/放置动画、进食/睡眠动画的完整落地（属其它状态机子状态，非本 PT 话题）。  
   - 任何 **未在 oh-gen-doc 中出现** 的验收辅助 UI。

## 验收标准

- [x] 小人执行 **开采** 读条时：可见 **开采** 相关动画表现 + **目标岩石上方** 进度条 + **开采中** 状态图标（或等价清晰语义）。  
- [x] 小人执行 **伐木** 读条时：可见 **伐木** 相关动画表现 + **目标树木上方** 进度条 + **伐木中** 状态图标（或等价清晰语义）。  
- [x] 非上述读条阶段不显示对应工作进度条；工作结束或中断后进度条与图标状态正确收回。  
- [x] `npm run build` 无编译错误。

## 与前后计划的关系

- **紧接 PT008**：补上规格中的 **表现层**，便于人工验收「执行工作状态」与 **UI系统** 进度/状态约定。  
- 与 **PT011（寻路）** 正交；若排期上需优先「看得见在工作」，本 PT 应 **早于或并行于** 非表现类优化，由调度按里程碑决定。
