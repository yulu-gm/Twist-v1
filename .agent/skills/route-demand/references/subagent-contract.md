# SubAgent 契约

## 主 agent 下发内容

每个 SubAgent 的输入必须包含：

- 原始需求摘要
- 本系统负责的玩家可见结果
- 本系统标准文档路径
- aidoc 输出路径
- 本系统默认 failing test 层级
- 已知 fake/stub 边界

## SubAgent 必须执行

1. 阅读系统标准文档。
2. 只提炼本系统需要承诺的行为和边界。
3. 回写一份系统 aidoc。
4. 标注：
   - 前置依赖
   - 本系统输入
   - 本系统输出/反馈
   - 假实现边界
   - 最先失败的测试
   - 最小通过实现
   - 后续反推到底层的接口/规则

## SubAgent 禁止行为

- 不要替其他系统补规格。
- 不要擅自改写系统注册表。
- 不要越过主 agent 直接决定跨系统接口。
- 不要把未确认的领域规则伪装成既定事实。

## 返回要求

SubAgent 的最终返回必须至少说明：

- 写入了哪个 aidoc 文件
- 本系统的首个 failing test 应该测什么
- 还依赖主 agent 汇总哪些外部系统结果
