# 编码规范

语言与工具：

- 使用 TypeScript。
- 优先采用 ES modules。
- 使用 Phaser 负责渲染与场景编排。
- 让模拟层与领域层代码尽量保持轻框架依赖。

代码风格：

- 优先使用语义明确的命名，而不是缩写。
- 保持函数简短且职责单一。
- 游戏状态优先使用普通对象与显式接口表达。
- 对有限集合的游戏概念使用枚举或字面量联合类型。
- 在早期脚手架阶段避免过早优化。

架构指导：

- 在可能的情况下拆分以下关注点：
  - 世界状态
  - 实体状态
  - AI 目标与工作选择
  - 行动执行
  - 渲染与 UI
- 角色行为应建模为：围绕需求、优先级与可执行世界行动进行目标驱动选择。
- 随机行为应可注入，以便测试可以稳定控制结果。

测试指导：

- 为工具函数与模拟规则补充测试。
- 任何确定性逻辑上的缺陷修复，在条件允许时都应加入回归测试。
- 现阶段优先快速单元测试，而不是缓慢的端到端测试。
