# 任务工作流

Agent 默认工作流：

1. 阅读 `Agent.md` 以及 `.agent/` 下的相关文件。
2. 在提出架构改动前先检查本地代码与文档。
3. 对于非简单功能，先在 `docs/ai/` 中创建或更新规格说明。
4. 以小步提交的方式实现。
5. 为确定性行为补充或更新测试。
6. 当初始化方式或协作流程变化时，同步更新面向人的文档。

功能开发工作流：

1. 明确玩法意图。
2. 明确模拟实体与状态迁移。
3. 明确角色 AI 如何评估目标并选择工作。
4. 先实现领域逻辑，再做较重的 UI 打磨。
5. 在 Phaser 场景中暴露对应行为。
6. 通过测试验证，并补上一段简短文档说明。

缺陷修复工作流：

1. 复现或隔离问题。
2. 确认问题发生在模拟层还是表现层。
3. 在最小且正确的层级完成修复。
4. 条件允许时加入回归测试。
5. 记录任何值得说明的规则变化。
