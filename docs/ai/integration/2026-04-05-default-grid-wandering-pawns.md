## 主题

`2026-04-05-default-grid-wandering-pawns`

## 玩家路径

1. 玩家打开游戏，场景初始化时读取默认地图配置与默认角色配置。
2. `world-grid` 优先提供地图尺寸、格子中心坐标以及默认出生点。
3. `pawn-state` 根据出生点创建五个角色状态，并且将名字与显示位置暴露给场景层。
4. 场景层绘制网格、圆形角色以及头顶名字。
5. `task-planning` 为每一个空闲角色挑选一个合法的相邻格子。
6. `pawn-state` 进入移动中状态，场景层按照插值进度将角色从当前格子中心平滑移动到目标格子中心。
7. 角色到达目标格子之后再次进入下一轮游荡选择。

## 参与系统

- `world-grid`：定义地图尺寸、格子坐标、相邻格子以及默认出生点。
- `pawn-state`：定义角色名字、逻辑格子位置、移动中状态以及显示位置。
- `task-planning`：定义唯一行为“游荡”及其选择格子规则。

## 当前用户界面优先的假实现

- 地图配置优先使用固定尺寸、固定格子大小、无障碍地图。
- 角色外观优先统一使用圆形与固定颜色，不引入正式美术资源。
- 游荡决策优先使用简单的合法相邻格子选择，不引入需求评估、任务队列以及寻路算法。

## 测试驱动开发顺序

1. 优先编写 `world-grid` 的 `domain` 测试，锁定默认出生点、相邻格子以及边界规则。
2. 其次编写 `pawn-state` 的 `domain` 测试，锁定默认角色快照以及显示位置插值。
3. 最后编写 `task-planning` 的 `domain` 测试，锁定合法相邻格子选择与等待分支。
4. 上述纯逻辑测试通过之后，再使用场景层将网格、圆形角色以及名字接入展示。

## 从假实现到真实实现反推顺序

1. 优先将硬编码的地图配置替换为 `data/` 目录下的地图定义。
2. 其次将硬编码的角色外观与出生点替换为正式角色数据。
3. 最后将简单的游荡规则替换为更加完整的目标评估与任务规划。

## 必须运行的回归测试组合

- `world-grid` + `pawn-state`
- `pawn-state` + `task-planning`
- `world-grid` + `task-planning`
