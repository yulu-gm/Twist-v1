## 主题

`2026-04-05-default-grid-wandering-pawns`

## 玩家路径

1. 玩家打开游戏，场景初始化时读取默认地图配置与默认角色配置。
2. `world-grid` 先提供地图尺寸、格子中心坐标和默认出生点。
3. `pawn-state` 根据出生点创建 5 个角色状态，并把名字与显示位置暴露给场景层。
4. 场景层绘制 grid、圆形角色和头顶名字。
5. `task-planning` 为每个空闲角色挑选一个合法邻格。
6. `pawn-state` 进入移动中状态，场景层按插值进度把角色从当前格中心平滑移动到目标格中心。
7. 角色到达目标格后再次进入下一轮游荡选择。

## 参与系统

- `world-grid`：定义地图尺寸、格坐标、邻格和默认出生点
- `pawn-state`：定义角色名字、逻辑格子位置、移动中状态和显示位置
- `task-planning`：定义唯一行为“游荡”及其选格规则

## 当前 UI-first fake

- 地图配置先用固定尺寸、固定格子大小、无障碍地图。
- 角色外观先统一用圆形和固定颜色，不引入正式美术资源。
- 游荡决策先用简单合法邻格选择，不引入需求评估、任务队列和寻路。

## TDD 顺序

1. 先写 `world-grid` 的 `domain` 测试，锁定默认出生点、邻格和边界规则。
2. 再写 `pawn-state` 的 `domain` 测试，锁定默认角色快照和显示位置插值。
3. 最后写 `task-planning` 的 `domain` 测试，锁定合法邻格选择与等待分支。
4. 上述纯逻辑转绿后，再用场景层把 grid、圆形角色和名字接出来。

## fake-to-real 反推顺序

1. 先把写死地图配置替换为 `data/` 下的地图定义。
2. 再把写死角色外观与出生点替换为正式角色数据。
3. 最后把简单游荡规则替换为更完整的目标评估和任务规划。

## 必跑回归组合

- `world-grid` + `pawn-state`
- `pawn-state` + `task-planning`
- `world-grid` + `task-planning`
