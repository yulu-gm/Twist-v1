## 目标

为默认开局提供一张可以直接运行的格子地图，并且给角色系统提供统一的格子坐标、相邻格子和出生点规则。

## 本系统负责的玩家可见结果

- 玩家打开游戏之后立即看到一张默认的 grid 地图。
- 地图尺寸稳定，格子线清晰，角色会对齐到格子中心。
- 地图边界之外不会被当作可走区域。

## 前置依赖

- 无。该系统是本次原型的底层前置系统。

## 本系统输入

- 默认地图宽度与高度
- 默认格子像素尺寸
- 默认出生点列表
- 角色当前位置占用信息

## 本系统输出/反馈

- 地图列数、行数和格子大小
- 格子中心点坐标换算结果
- 某个格子的正交相邻格子列表
- 可行走判断和占用判断
- 供五个角色使用的默认出生点集合

## 假实现边界

- 允许首先使用固定宽度与高度、固定格子大小、无障碍的地图。
- 允许首先使用写死的五个出生点，不接入地图编辑器或者 JSON 内容管线。
- 不允许在 Phaser 场景里面直接写死另一套相邻格子规则。

## 最先失败的测试

- 测试层级：`domain`
- 触发方式：请求默认地图配置的出生点和相邻格子列表
- 预期失败原因：项目中还没有统一的格子地图定义，也没有相邻格子与边界判断

## 最小通过实现

- 定义默认地图配置对象
- 提供格子中心换算
- 提供边界内部正交相邻格子枚举
- 提供默认出生点列表和占用查询

## 后续反推到底层的接口/规则

- 当前写死的地图将来可以替换为 `data/` 下的地图配置文件
- 占用判断将来可以扩展为地形阻挡、建筑占据格子和动态寻路成本
- 替换之后需要补充地图加载与阻挡规则的 `domain` 回归测试
