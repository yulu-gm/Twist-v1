## 目标

为默认开局提供一张可直接运行的格子地图，并给角色系统提供统一的格子坐标、邻格和出生点规则。

## 本系统负责的玩家可见结果

- 玩家打开游戏后立即看到一张默认 grid 地图。
- 地图尺寸稳定，格线清晰，角色会对齐到格子中心。
- 地图边界之外不会被当成可走区域。

## 前置依赖

- 无。该系统是本次原型的底层前置系统。

## 本系统输入

- 默认地图宽高
- 默认格子像素尺寸
- 默认出生点列表
- 角色当前位置占用信息

## 本系统输出/反馈

- 地图列数、行数和格子大小
- 格子中心点坐标换算结果
- 某格的正交邻格列表
- 可行走判断和占用判断
- 供 5 个角色使用的默认出生点集合

## 假实现边界

- 允许先用固定宽高、固定格子大小、无障碍地图。
- 允许先用写死的 5 个出生点，不接地图编辑器或 JSON 内容管线。
- 不允许在 Phaser 场景里直接写死另一套邻格规则。

## 最先失败的测试

- 测试层级：`domain`
- 触发方式：请求默认地图配置的出生点和邻格列表
- 预期失败原因：项目中还没有统一的格子地图定义，也没有邻格与边界判断

## 最小通过实现

- 定义默认地图配置对象
- 提供格子中心换算
- 提供边界内正交邻格枚举
- 提供默认出生点列表和占用查询

## 后续反推到底层的接口/规则

- 当前写死地图将来可替换为 `data/` 下的地图配置文件
- 占用判断将来可扩展为地形阻挡、建筑占格和动态寻路成本
- 替换后需要补地图加载与阻挡规则的 `domain` 回归测试
