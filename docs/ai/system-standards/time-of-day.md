# time-of-day 系统文档规范

## 系统职责

负责世界时间推进、跨天归一化、时间格式化，以及供场景层消费的昼夜视觉调色板采样。

## 玩家可见结果

- 玩家能感知到世界时间持续流逝，而不是停留在静态场景。
- 场景背景、格线和文字会随着时间平滑变化，不出现突兀跳色。
- HUD 能稳定展示当前天数与具体时间。

## 输入与输出边界

- 输入：真实时间增量、起始时间配置、昼夜关键帧调色板。
- 输出：当前天数、当前时刻、格式化时间文本、当前视觉调色板。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认 failing test

优先写 `domain` 测试，验证时间推进、跨天归一化、格式化与关键帧插值的确定性。

## fake/stub 规则

允许：

- 先使用写死的关键时间点与调色板锚点驱动场景表现。

不允许：

- 在 Phaser 场景里直接拼装昼夜推进规则，绕过独立领域模块。
