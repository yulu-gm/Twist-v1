# selection-ui 系统文档规范

## 系统职责

负责玩家对对象的选中、焦点切换、可交互高亮以及与选中状态直接相关的即时反馈。

## 玩家可见结果

- 玩家清楚当前选中了哪个实体或内容。
- 玩家能够辨识哪些对象处于可选、不可选或已切换的状态。
- 选中状态的改变会立即驱动其他展示系统进行刷新。

## 输入与输出边界

- 输入：点击、框选、快捷键切换、场景中的可选对象集合。
- 输出：当前选中目标、可视化高亮、选中变更事件。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出或者反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口或者规则

## 默认失败的测试

优先编写 `acceptance` 测试，验证玩家操作之后是否呈现正确的选中反馈。

## 伪造与存根规则

允许：

- 使用固定的对象列表来驱动选中逻辑。
- 使用硬编码的可选状态来替代真实的规则。

不允许：

- 将真实的选中来源与用户界面高亮逻辑混合编写在一个不可替换的庞大对象之中。
