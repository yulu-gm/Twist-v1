# selection-ui 系统文档规范

## 系统职责

负责玩家对对象的选中、焦点切换、可交互高亮以及与选中状态直接相关的即时反馈。

## 玩家可见结果

- 玩家知道当前选中了谁或什么。
- 玩家能看出哪些对象可选、不可选、已切换。
- 选中变化会立刻驱动其他展示系统刷新。

## 输入与输出边界

- 输入：点击、框选、快捷键切换、场景中的可选对象集合。
- 输出：当前选中目标、可视化高亮、选中变更事件。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认 failing test

优先写 `acceptance` 测试，验证玩家操作后是否出现正确的选中反馈。

## fake/stub 规则

允许：

- 用固定对象列表驱动选中
- 用写死的可选状态代替真实规则

不允许：

- 把真实选中来源和 UI 高亮混写在一个不可替换的巨型对象里
