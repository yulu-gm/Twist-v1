# scene-hud 系统文档规范

## 系统职责

负责 HUD、状态卡、菜单、按钮、提示条和场景内的附加信息展示。

## 玩家可见结果

- 玩家能从界面读到当前对象状态和下一步可操作项。
- 面板开关、显隐、禁用态和提示信息足够清晰。

## 输入与输出边界

- 输入：选中事件、场景状态、角色可读状态、可执行动作列表。
- 输出：面板内容、按钮状态、信息文案、局部过渡效果。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认 failing test

优先写 `component` 测试；如果涉及明确的人机交互路径，可升级为 `acceptance` 测试。

## fake/stub 规则

允许：

- 使用 fixture 状态卡数据
- 用假按钮可用性代替真实规则检查

不允许：

- 把领域计算直接塞进 HUD 组件
