# scene-hud 系统文档规范

## 系统职责

负责平视显示器、状态卡片、菜单、按钮、提示条以及场景内部的附加信息展示。

## 玩家可见结果

- 玩家能够从界面读取到当前对象的状态以及下一步的可操作选项。
- 面板的开启与关闭、显示与隐藏、禁用状态以及提示信息必须足够清晰。

## 输入与输出边界

- 输入：选中事件、场景状态、角色可读状态、可执行动作列表。
- 输出：面板内容、按钮状态、信息文案、局部过渡效果。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出或者反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口或者规则

## 默认失败的测试

优先编写 `component` 测试；如果涉及明确的人机交互路径，可以升级为 `acceptance` 测试。

## 假实现与存根规则

允许：

- 使用测试夹具提供的状态卡片数据。
- 使用虚假的按钮可用性来替代真实的规则检查。

不允许：

- 将领域计算逻辑直接放入平视显示器组件之中。
