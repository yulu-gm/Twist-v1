# pawn-state 系统文档规范

## 系统职责

负责角色的可读状态模型，包括属性、需求、状态标签和供 UI/AI 消费的派生字段。

## 玩家可见结果

- 玩家看到的角色状态稳定、可解释、不会在不同界面中自相矛盾。

## 输入与输出边界

- 输入：基础属性、需求值、状态效果、时间推进或任务结果。
- 输出：角色状态快照、状态标签、可供 HUD 和规划系统消费的派生字段。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认 failing test

优先写 `domain` 测试，验证状态快照和派生字段的确定性。

## fake/stub 规则

允许：

- 在 UI-first 阶段用固定状态快照喂给上层系统

不允许：

- 让多个界面各自拼接一套不同的角色状态语义
