# pawn-state 系统文档规范

## 系统职责

负责角色的可读状态模型，包括属性、需求、状态标签以及供用户界面与人工智能读取的派生字段。

## 玩家可见结果

- 玩家观察到的角色状态保持稳定、具备可解释性，并且不会在不同的界面之中出现自相矛盾的情况。

## 输入与输出边界

- 输入：基础属性、需求数值、状态效果、时间推进或者任务结果。
- 输出：角色状态快照、状态标签、可供平视显示器与规划系统读取的派生字段。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出或者反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口或者规则

## 默认失败的测试

优先编写 `domain` 测试，验证状态快照与派生字段的确定性。

## 假实现与存根规则

允许：

- 在用户界面优先阶段，使用固定的状态快照提供给上层系统。

不允许：

- 让多个界面各自拼接出一套不同的角色状态语义。
