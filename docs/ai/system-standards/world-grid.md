# world-grid 系统文档规范

## 系统职责

负责格子地图的静态尺寸、格子坐标系、邻近格子规则、可行走边界、默认出生点和占用快照。

## 玩家可见结果

- 玩家看到一个稳定、可读的格子地图。
- 地图上的角色始终落在格子逻辑上，不会出现“站在半个格子里”的规则错误。
- 地图上可出现静态阻挡（例如石头占位）；可走判断与规划必须一致地尊重阻挡语义。
- 后续所有按格子移动、寻路和占格子判断都基于同一套地图语义。

## 输入与输出边界

- 输入：地图尺寸、格子大小、阻挡信息、默认出生配置、角色占用更新。
- 输出：格子坐标换算、邻近格子列表、可行走判断、占用查询、默认出生点集合。

## 人工智能文档必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出与反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口与规则

## 默认失败测试

优先写 `domain` 测试，验证邻近格子、边界和默认出生点的确定性。

## 伪造实现与存根规则

允许：

- 先用固定地图尺寸和写死阻挡为空的配置驱动原型
- 在配置中提供 `blockedCellKeys`（或等价结构），阻挡可由运行时随机抽样或未来由 `data/` 加载
- 先用固定出生点列表代替正式地图生成流程

不允许：

- 让场景渲染层私自定义另一套格子坐标语义
- 让角色移动逻辑绕过地图可行走规则直接修改坐标
