# task-planning 系统文档规范

## 系统职责

负责目标评估、工作选择、计划生成和动作候选排序。

## 玩家可见结果

- 玩家能从角色行为或 UI 提示中感受到当前计划是如何被挑选出来的。
- 角色移动计划不得违反地图可走规则（含阻挡格）。

## 输入与输出边界

- 输入：角色状态、世界状态、优先级、可执行动作集合。
- 输出：任务候选、当前计划、计划原因摘要、拒绝原因。

## aidoc 必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认 failing test

优先写 `domain` 测试，验证规则、排序和状态迁移。

## fake/stub 规则

允许：

- UI-first 阶段用写死的任务候选列表驱动上层界面

不允许：

- 在规划系统里直接依赖 Phaser 场景对象
