# world-core 系统文档规范

## 系统职责

负责世界真相源、统一实体标识、可序列化只读快照、格子占用索引，以及标记/蓝图到工作单与建造结果的闭环。

## 玩家可见结果

- 玩家触发标记或建造之后，世界中的实体、占格和结果不会彼此打架。
- 同一格子的占用冲突会得到稳定且可预测的处理结果。
- 时间推进、暂停、调速、昼夜时段、工作领取、工作失败和建造落地都能从同一份世界快照中读取。
- 建筑完成后，会生成后续系统可消费的只读结果，例如床位休息点。

## 输入与输出边界

- 输入：地图配置、时间配置、实体增删改命令、标记命令、蓝图命令、工作领取/完成/失败命令。
- 输出：世界时间快照、时间事件、实体快照、占用索引、工作单快照、标记快照、建造产物、派生结果。

## 人工智能文档必填字段

- 目标
- 本系统负责的玩家可见结果
- 前置依赖
- 本系统输入
- 本系统输出/反馈
- 假实现边界
- 最先失败的测试
- 最小通过实现
- 后续反推到底层的接口/规则

## 默认失败测试

优先写 `domain` 测试，验证实体/占用一致性、快照可序列化、暂停时无隐式推进、时间控制状态、工作锁定与去重、工作失败重试，以及蓝图建造落地。

## 伪造实现与存根规则

允许：

- 先用纯领域命令回放驱动标记、工作和建造闭环，不要求先接入完整 UI。
- 先只支持有限的工作种类与建筑种类，例如障碍拆除和床铺建造。
- 先用只读派生结果暴露建筑后效，而不是提前接通完整 AI 消费链路。

不允许：

- 让场景层或 UI 层私自维护另一份实体真相源。
- 绕过占用检查直接写入实体位置。
- 让工作锁定状态散落在多个系统副本中。
- 对外直接暴露内部 `Map` 或可回写引用，破坏快照的可序列化与只读语义。
